# 基金研究团队模型展示平台 — 整体需求文档

## 一、项目背景

基金研究团队希望将自身研究成果对外展示，构建一个面向客户的 Web 平台。平台包含两大核心能力：

1. **模型展示系统**：以 Dashboard / 报告形式展示各类金融模型的结果，支持交互筛选和数据下载
2. **AI 问答机器人**：智能对话入口，可根据用户提问自动推荐相关模型页面，引导用户深入探索

两者通过 AI 的 Function Calling 能力联动——AI 在回答中结构化返回推荐的模型页面，前端渲染为可点击的链接卡片。

---

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Vite | SPA，组件化开发，适合后端背景团队上手 |
| 后端 | FastAPI | 异步框架，提供数据 API + AI 对话接口 |
| 数据库 | Doris（生产） / MySQL（本地开发） | Doris 兼容 MySQL 语法，开发环境用 Docker MySQL 模拟 |
| 大模型 | 待定（OpenAI / Claude / 本地模型） | 通过后端中转调用，支持 Function Calling |
| 通信 | SSE（Server-Sent Events） | AI 回答流式输出（打字机效果） |

### 开发环境策略

- **本地开发**：Docker 运行 MySQL 8.0 容器，导入 Doris 数据子集进行开发测试
- **生产环境**：直连 Doris 数据库
- **切换方式**：后端 config.py 通过环境变量区分 dev / prod，SQL 语句通用

```bash
# 本地启动 MySQL
docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=dev mysql:8.0
```

## 三、模型展示系统

### 3.1 整体设计原则

- **插件化 / 解耦架构**：每个模型页面作为独立模块，可自由新增、移除、排列
- **组件复用**：图表卡片、筛选器、数据表格等封装为通用组件，各模型页面按需组合
- **统一导航**：首页汇总所有可用模型入口，支持灵活配置展示哪些页面

### 3.2 模型页面功能

- **展示形式**：以 Dashboard 为主（多图表 + 关键指标），部分模型可能以报告形式呈现
- **图表类型**：折线图、柱状图、表格、热力图等，视具体模型需求而定
- **交互能力**：
  - 日期范围选择
  - 参数切换 / 筛选
  - 数据下载（导出 Excel）
- **初始规模**：3–4 个模型页面，后续持续扩展

### 3.3 数据更新频率

| 类型 | 频率 | 示例 |
|------|------|------|
| 日频数据 | 每日更新 | 市场指标、模型信号 |
| 定期报告 | 季度 / 不定期 | 策略报告、回测结果 |

数据通过后端定时任务写入数据库，前端请求时从数据库读取（非实时计算）。

## 四、AI 问答机器人

### 4.1 聊天界面

- 对话输入框 + 发送按钮
- 消息气泡展示（用户 / AI 区分左右）
- AI 回答支持 Markdown 渲染（代码块、表格、加粗等）
- 流式输出：逐字显示，非等全部生成完再展示
- 自动滚动到最新消息

### 4.2 后端 API

```
POST /api/chat
```

请求体：

```json
{
  "message": "用户输入的问题",
  "history": [
    {"role": "user", "content": "之前的问题"},
    {"role": "assistant", "content": "之前的回答"}
  ]
}
```

响应：SSE 流式返回，每个 chunk 格式：

```
data: {"content": "一段", "done": false}
data: {"content": "文字", "done": false}
data: {"content": "", "done": true}
```

### 4.3 AI → 模型页面联动（Function Calling 方案）

AI 在回答时可调用 `recommend_model` tool，返回结构化数据：

```json
{
  "tool": "recommend_model",
  "model_id": "yield-curve",
  "display_name": "收益率曲线模型",
  "reason": "您可以查看最新的收益率曲线走势"
}
```

前端接收后渲染为可点击的链接卡片，跳转至对应模型页面。

**优势**：相比占位符方案，不依赖正则解析，格式稳定，扩展性好。

### 4.4 后端逻辑

- 接收前端消息 + 历史对话
- 拼接 prompt + tools 定义，调用大模型 API
- 流式转发大模型输出到前端
- 解析 Function Calling 结果，附带模型链接信息
- 基础错误处理（API 超时、key 失效等）

## 五、部署架构

```
用户（公网）
  ↓ HTTPS
反向代理（Nginx）
  ↓
内网服务器
  ├── 前端静态文件（Nginx 托管 / Vite build 产物）
  ├── FastAPI 后端（Gunicorn + Uvicorn workers）
  └── Doris 数据库
```

- 网站部署在内网，通过反向代理暴露到公网
- 已有 HTTPS 域名
- 前端打包后为纯静态文件，由 Nginx 直接托管

## 七、开发计划

### Phase 1 — AI 问答机器人 Demo（当前）

- 搭建 Vue 3 + Vite 项目脚手架
- 实现聊天界面（输入、气泡、Markdown 渲染、流式输出）
- FastAPI 后端包装大模型 API，SSE 流式返回
- 跑通完整对话链路

### Phase 2 — 模型展示页面

- 搭建路由和导航框架
- 实现通用图表 / 表格 / 筛选组件
- 接入第一个模型 Dashboard
- 实现 Excel 导出功能

### Phase 3 — AI 联动 + 扩展

- 接入 Function Calling，AI 回答嵌入模型链接卡片
- 扩展更多模型页面
- 用户权限控制（可选）
- 对话持久化存储（可选）
